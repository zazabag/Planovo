ВАЖНО
-----
Этот файл — ориентир для разработки и продуктовой команды, НЕ юридическая
консультация. Перед публикацией документов и запуском формы согласуйте тексты
с юристом под вашу организационно-правовую форму (ИП / ООО), фактический
состав данных, интеграции (CRM, почта, аналитика) и модель монетизации.

Нормативная база (основное)
---------------------------
[ ] Учтены требования ФЗ № 152-ФЗ «О персональных данных»
[ ] Учтены требования ФЗ № 149-ФЗ «Об информации…»
[ ] Учтены требования ЗоЗПП (ФЗ № 2300-1) — если есть дистанционная продажа
[ ] Учтены требования ФЗ № 38-ФЗ «О рекламе» — если есть рассылки / реклама
[ ] Учтены разъяснения Роскомнадзора по cookie (в т.ч. письмо № 07-об/дги-11030
    от 30.08.2023) и практика по равнозначности «принять / отклонить»
[ ] Учтены изменения с 01.09.2025: согласие на обработку ПДн — отдельный
    документ, не «вшитое» в оферту / политику / пользовательское соглашение
[ ] Учтена локализация: с 01.07.2025 первичный сбор и хранение ПДн граждан РФ
    — на территории РФ (ч. 5 ст. 18 152-ФЗ), если применимо к вашей схеме

Кто становится «оператором»
---------------------------
Сайт с лид-формой (имя, телефон, email, организация, комментарий) = сбор ПДн.
Даже 1–2 поля делают владельца сайта оператором персональных данных.

ЧАСТЬ 1. ДОКУМЕНТЫ НА САЙТЕ (СТРАНИЦЫ / PDF)
================================================================================

1.1. ОБЯЗАТЕЛЬНО — Политика обработки персональных данных
---------------------------------------------------------
(часто называют «Политика конфиденциальности»; юридически корректнее —
«Политика в отношении обработки персональных данных»)

[ ] Отдельная страница, постоянный URL (например: /privacy.html)
[ ] Ссылка доступна с любой страницы (футер, меню)
[ ] Указан оператор: наименование, ИНН/ОГРН, юр. адрес, email для обращений
[ ] Перечень обрабатываемых ПДн (для лид-формы: ФИО, телефон, email, название
    организации, должность, иные поля формы)
[ ] Цели обработки (обработка заявки, обратная связь, демо, договор и т.д.)
[ ] Правовые основания (согласие субъекта; при необходимости — иные основания)
[ ] Способы и сроки обработки, хранения, уничтожения
[ ] Передача третьим лицам / поручение обработки (хостинг, CRM, почта, мессенджеры)
[ ] Трансграничная передача — если есть (лучше минимизировать для РФ-граждан)
[ ] Права субъекта: доступ, уточнение, блокирование, удаление, отзыв согласия
[ ] Порядок обращений субъекта и сроки ответа
[ ] Меры защиты ПДн (общее описание)
[ ] Использование cookie и иных идентификаторов (отдельный раздел или ссылка
    на Cookie Policy)
[ ] Дата редакции документа

1.2. ОБЯЗАТЕЛЬНО — Согласие на обработку персональных данных (отдельный документ)
---------------------------------------------------------------------------------
С 01.09.2025 нельзя «прятать» согласие внутри оферты, политики или текста
«нажимая Отправить, вы соглашаетесь…» без отдельной страницы согласия.

[ ] Отдельная страница (например: /consent-pdn.html)
[ ] В тексте: кто оператор, какие данные, цели, срок действия согласия,
    порядок отзыва, ссылка на политику
[ ] Для лид-формы — отдельное согласие под цель «обработка заявки / обратная
    связь» (не смешивать с рассылкой рекламы)

1.3. ОБЯЗАТЕЛЬНО ДЛЯ ФОРМ — UI-согласие (чекбокс) + ссылка на документ
-----------------------------------------------------------------------
[ ] Под каждой формой сбора ПДн — чекбокс (НЕ включён по умолчанию)
[ ] Текст вида: «Даю согласие на обработку персональных данных в соответствии
    с [Согласием на обработку ПДн] и [Политикой]» — со ссылками
[ ] Кнопка отправки заблокирована, пока чекбокс не отмечен
[ ] Нельзя отправлять форму «молча» одним кликом без активного действия
[ ] Сохраняется факт согласия: дата/время, версия документа, IP (если нужно
    для доказательства — на стороне backend/CRM)

1.4. COOKIE / АНАЛИТИКА — баннер и политика
------------------------------------------
Cookie и похожие технологии (localStorage для трекинга, пиксели, Метрика,
GA, чаты, виджеты) при сборе идентификаторов пользователя относятся к
обработке данных и требуют информирования и согласия (кроме строго
необходимых технических cookie).

[ ] Баннер при первом визите (до загрузки необязательных скриптов)
[ ] Кнопки равнозначной заметности: «Принять все» / «Отклонить все»
    (или эквивалент; не прятать отказ)
[ ] Опционально: «Настроить» по категориям (необходимые / аналитика / маркетинг)
[ ] До согласия НЕ грузить: Яндекс.Метрику, Google Analytics, рекламные
    пиксели, сторонние чаты с трекингом
[ ] Отдельная Cookie Policy или раздел в политике ПДн: перечень cookie,
    цели, сроки, как отозвать согласие
[ ] Запись выбора пользователя (cookie consent log) — желательно

Технические cookie (сессия, CSRF, язык, корзина) — обычно без отдельного
согласия, но их нужно описать в политике.

1.5. РЕКЛАМА И РАССЫЛКИ — отдельное согласие (если планируется)
---------------------------------------------------------------
Если лид-форма подразумевает «присылайте новости / акции»:

[ ] Отдельный чекбокс (не обязательный для отправки заявки)
[ ] Отдельный документ или раздел: согласие на получение рекламно-информационных
    материалов (ФЗ о рекламе, 152-ФЗ)
[ ] Возможность отписаться в каждом письме

1.6. ПУБЛИЧНАЯ ОФЕРТА — когда нужна, когда нет
-----------------------------------------------
Оферта НЕ заменяет политику ПДн и НЕ заменяет согласие на обработку ПДн.

ОФЕРТА НУЖНА (или иной договор), если:
[ ] На сайте продаются товары/услуги дистанционно
[ ] Есть онлайн-оплата, подписка, тарифы с автоматическим заключением договора
[ ] Пользователь оформляет заказ с конкретными условиями (цена, срок)

ДЛЯ ПРОСТОЙ ЛИД-ФОРМЫ («оставьте заявку, мы перезвоним») оферта часто
НЕ обязательна, если нет автоматического заключения договора на сайте.
Но при продаже SaaS-подписки Planovo оферта понадобится на этапе оплаты.

[ ] Публичная оферта на отдельной странице (если есть продажи)
[ ] Пользовательское соглашение / условия использования сервиса (для
    зарегистрированных пользователей продукта — отдельно от лид-формы)

1.7. РЕКВИЗИТЫ И ОБЯЗАТЕЛЬНАЯ ИНФОРМАЦИЯ О КОМПАНИИ
---------------------------------------------------
[ ] В футере: наименование юрлица / ИП, ИНН, ОГРН (или ОГРНИП)
[ ] Контактный email (и телефон, если заявлен)
[ ] Ссылка на политику ПДн
[ ] Ссылка на согласие ПДн
[ ] Ссылка на cookie-политику (если используются необязательные cookie)

Для ИП/ООО дополнительно могут требоваться сведения по иным нормам (не все
размещаются на лендинге) — уточнить у юриста.

================================================================================
ЧАСТЬ 2. ЛИД-ФОРМА НА ГЛАВНОЙ (Planovo) — МИНИМУМ ДЛЯ ЗАПУСКА
================================================================================

Поля формы (пример — согласовать с продуктом)
[ ] Имя
[ ] Телефон или Email (хотя бы один контакт — ПДн)
[ ] Организация / тип клиента (школа, клуб, секция) — опционально
[ ] Комментарий — опционально

На странице с формой
[ ] Чекбокс согласия на обработку ПДн (обязательный)
[ ] Ссылки на Политику и текст Согласия открываются в новой вкладке
[ ] Отдельный чекбокс на рекламную рассылку — только если реально планируется
[ ] Подпись под кнопкой: «Нажимая «Отправить», вы подтверждаете…» — НЕ вместо
    чекбокса, а дополнительно (после 01.09.2025 основа — отдельное согласие)
[ ] Сообщение об успешной отправке без публикации чужих ПДн
[ ] HTTPS на боевом домене

Техническая обработка заявки (не видно пользователю, но для законности)
[ ] Данные формы хранятся / обрабатываются с учётом локализации (РФ)
[ ] Договоры с подрядчиками (почта, CRM, хостинг) — поручение обработки ПДн
[ ] Ограничен доступ к заявкам, срок хранения, удаление по запросу
[ ] Логирование: когда отправлена заявка, какая версия согласия принята

ЧАСТЬ 3. НЕ НА САЙТЕ, НО ОБЯЗАТЕЛЬНО ДЛЯ ОПЕРАТОРА (152-ФЗ)
================================================================================

Эти пункты не размещаются как баннер, но без них сбор данных формально
незаконен:

[ ] Уведомление Роскомнадзора об обработке ПДн (если нет исключений по ч. 2
    ст. 22 152-ФЗ) — через pd.rkn.gov.ru
[ ] Приказ / назначение ответственного за организацию обработки ПДн (для юрлиц)
[ ] Внутренние документы: положение об обработке ПДн, регламенты, модель угроз
    (объём зависит от масштаба)
[ ] Реестр обработки, журнал обращений субъектов
[ ] Договоры поручения с теми, кто обрабатывает данные по вашему поручению
[ ] Оценка необходимости уведомления о трансграничной передаче
[ ] Обучение сотрудников, имеющих доступ к заявкам

ЧАСТЬ 4. ТИПИЧНЫЕ ОШИБКИ (ПРОВЕРИТЬ ПЕРЕД РЕЛИЗОМ)
================================================================================

[ ] Галочка согласия включена по умолчанию
[ ] Нет ссылки на политику / текст согласия
[ ] Согласие на ПДн спрятано только в оферте или только в политике
[ ] Метрика/пиксели грузятся до cookie-баннера
[ ] Форма уходит на зарубежный сервис без локализации и без поручения
[ ] В политике указаны цели, не совпадающие с реальностью (например, «аналитика»
    есть, а в политике не описана)
[ ] Нет способа отозвать согласие (email или форма обращения)
[ ] В футере нет реквизитов оператора
[ ] Один чекбокс «согласен со всем» на ПДн + рекламу + оферту

ЧАСТЬ 5. ПРИОРИТЕТ ДЛЯ PLANOVO (ЛЕНДИНГ + ЛИД-ФОРМА)
================================================================================

Сделать в первую очередь (MVP compliance)
-----------------------------------------
1. Страница «Политика обработки ПДн»
2. Страница «Согласие на обработку ПДн»
3. Футер со ссылками + реквизиты оператора
4. Лид-форма с обязательным чекбоксом и блокировкой отправки
5. Cookie-баннер — если подключите Метрику или иные трекеры

Сделать до включения оплаты / подписки
--------------------------------------
6. Публичная оферта (или договор SaaS)
7. Пользовательское соглашение для аккаунта в продукте
8. Политика возвратов / тарифов (если применимо)

Сделать на стороне бизнеса (параллельно с разработкой)
------------------------------------------------------
9. Уведомление в Роскомнадзор (при необходимости)
10. Выбор российского хостинга/CRM/почты для заявок
11. Договоры поручения с подрядчиками
12. Юридическая вычитка всех текстов
