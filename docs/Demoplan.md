# Demoplan — демо «визуальный пример + пояснения»

> Рабочий план для локальных правок `*.demo.js` / `*.html`.  
> Цель: за **20–30 секунд** показать, *как выглядит* продукт и *почему* им удобно пользоваться — без имитации полноценной системы.  
> Статус: **education ✅** · **sports ✅** · **clubs ✅**

---

## 1. Модель демо

Демо на лендинге — **не мини-ERP**, а **витрина**:

| Было | Стало |
|------|--------|
| 3 роли, много табов, KPI, live-таймеры | **Блочный тур**: пояснение слева → фрагмент UI справа |
| Пользователь «играет» в систему | Пользователь **смотрит** и читает короткие пояснения |
| Пояснения сверху / справа колонкой | **Комментарий слева**, пример **справа** (на узком экране — комментарий сверху) |
| Полная неделя в сетке | **Пн + Ср**, **2 слота** |
| 5–6 секций / групп | **2 варианта** для единственного клика |

### Макет страницы

```
┌──────────────────────────────────────────────────────────────┐
│  Шапка: логотип · вкладки ролей · «Вернуться на сайт»        │
├──────────────────────────────────────────────────────────────┤
│  Краткий intro (1 строка)                                    │
├──────────────────────────────────────────────────────────────┤
│  ┌─ Блок 1 ───────────────────────────────────────────────┐  │
│  │ [пояснение]  │  [фрагмент UI: день / расписание]      │  │
│  └────────────────────────────────────────────────────────┘  │
│  ┌─ Блок 2 ───────────────────────────────────────────────┐  │
│  │ [пояснение]  │  [переключатель секции / группы]        │  │
│  └────────────────────────────────────────────────────────┘  │
│  ┌─ Блок 3 ───────────────────────────────────────────────┐  │
│  │ [пояснение]  │  [мини-сетка Пн+Ср × 2 слота]           │  │
│  └────────────────────────────────────────────────────────┘  │
│  Подпись «В полной версии…» + вкладки ролей в шапке          │
└──────────────────────────────────────────────────────────────┘
     ░░░ фон страницы: doodle-паттерн (как в Telegram),        │
         прокручивается вместе с контентом                     │
```

### Принципы

1. **Визуальный пример** — карточки, сетка, типографика продукта.
2. **Пояснение слева** — зачем этот фрагмент и кому он нужен.
3. **Один сценарий по умолчанию** — ученик / клиент / участник.
4. **Один клик** — переключить группу или секцию (2 варианта).
5. **Роли в шапке** — упрощённые витрины тренера и админа (по 2 блока).
6. **Честная подпись** — «Визуальный пример · не полная система».

## 2. Общие технические решения

| Параметр | Значение |
|----------|----------|
| Сетка | **Пн + Ср** (индексы `0` и `2`), **2 слота** |
| Слоты (sports) | Утро + вечер (`0` и `6` в `TIME_SLOTS`) |
| Данные | Mock из `SCHEDULE_DATA` / `generateSchedule` |
| Роли | Персона по умолчанию + **2 вкладки** в шапке (упрощённые) |
| Интерактив | **0–1** переключатель (2 группы / 2 секции) |
| Пояснения | `*_SHOWCASE_BLOCKS[]` + `ShowcaseBlock` |
| Разметка | `.showcase-walkthrough` → `.showcase-block` (grid 300px \| 1fr) |
| Фон | `assets/*-doodle-pattern.svg` на `body`, **scroll** (не fixed) |
| CSS | `.showcase-block-note`, `.showcase-block-demo`, `.group-selector` |

```js
const DEMO_DAY_INDICES = [0, 2]; // Пн, Ср
const DEMO_SLOT_INDICES = [0, 6]; // 2 слота с тренировками
const DEMO_SLOTS = DEMO_SLOT_INDICES.map((i) => ({ index: i, ...TIME_SLOTS[i] }));
```

Компоненты (повторяются в каждом `*.demo.js`):

- `ShowcaseBlock`, `ShowcaseDemoChrome`
- `ShowcaseDayList` / `ShowcaseScheduleGrid` (адаптация под нишу)
- `*Showcase` — клиент; `CoachShowcase` / `AdminShowcase` — 2 блока с `showcaseFixedTab`

## 4. Sports — спортивные секции ✅

**Файлы:** `sports.html`, `sports.demo.js`  
**Цвет:** `#10b981` · **Фон:** `assets/sports-doodle-pattern.svg`  
**Статус:** витрина реализована

### Вкладка «Клиент» (3 блока)

| # | Пояснение | Пример справа |
|---|-----------|---------------|
| 1 | День тренировок | Понедельник: время, секция, группа, тренер, **занято/мест** |
| 2 | Своя секция | **Борьба / Фитнес** (единственный клик) |
| 3 | Фрагмент недели | Сетка Пн+Ср × 2 |

### Вкладка «Тренер» (2 блока)

| # | Пример |
|---|--------|
| 1 | Расписание секции (мини-сетка) |
| 2 | Посещаемость (отметка ✓/✗) |

Упрощено: 2 секции в селекторе, без «Следующая тренировка», без вкладки «Группы».

### Вкладка «Администрация» (2 блока)

| # | Пример |
|---|--------|
| 1 | Обзор — 3 KPI (клиенты, секции, абонементы) |
| 2 | Секции — карточки с загрузкой |

Упрощено: без отчётов и полного дашборда из 6 KPI.

### Не показывать в витрине

- 5 секций, каталог с записью и пробным занятием
- Live «сейчас идёт тренировка»
- Полная неделя (7 дней) в клиентском демо

## 8. Чего не делать

- Не возвращать полный мульти-ролевой ERP на первый экран.
- Не класть doodle-паттерн **внутрь** белых блоков — только **фон страницы**.
- Не использовать `background-attachment: fixed` (паттерн должен скроллиться с контентом).
- Не выгружать на GitHub без отдельного запроса.

## 9. Критерий готовности
