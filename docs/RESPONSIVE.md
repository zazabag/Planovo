# Planovo — адаптивный интерфейс

> **Источник истины** для вёрстки под разные экраны.  
> Цель: на любом разрешении контент **виден целиком** — без обрезки страницы и без «уезжания» блоков за край.  
> Реализация: `assets/site-mobile.css` (лендинг + legal), `assets/demo-mobile.css` (демо), базовые стили — `_next/static/chunks/ca02de87dd32ea73.css`, `assets/site-legal.css`, inline-стили в `*.html`.

---

## 1. Принципы (обязательные)

| # | Правило | Зачем |
|---|---------|--------|
| 1 | **Нет горизонтального скролла у `html`/`body`** | Страница не «ездит» влево-вправо |
| 2 | **Широкие таблицы и mockup — скролл внутри контейнера** | `.schedule-wrapper`, `.mockup-timetable` с `overflow-x: auto` |
| 3 | **`min-width: 0` у колонок grid/flex** | Иначе дочерние элементы раздувают родителя |
| 4 | **Не использовать жёсткий `min-width` у сеток на всю страницу** | Допустимо только внутри scroll-обёртки |
| 5 | **3D `transform` у mockup отключается ≤1024px** | Поворот выносит блок за границы viewport |
| 6 | **Touch-target ≥ 44×44px** | Кнопки, бургер, role-tab на мобильном |
| 7 | **Поля форм ≥ 16px на iOS (≤480px)** | Иначе Safari зумит при фокусе |
| 8 | **`env(safe-area-inset-*)`** | Вырез iPhone, home indicator |
| 9 | **Текст: `overflow-wrap: anywhere` / `word-break` в узких колонках** | Длинные слова не рвут layout |
| 10 | **При сомнении — одна колонка + вертикальный стек** | Лучше скролл вниз, чем вбок |

### Стратегия overflow

```
Страница (body)     → overflow-x: hidden | clip
Секция / container  → max-width: 100%, min-width: 0
Широкая сетка       → обёртка overflow-x: auto (скролл пальцем внутри карточки)
```

## 9. Как применять при разработке

1. **Новый блок на лендинге** — сначала строка в §4, затем стили в `site-mobile.css` (или базовый CSS при rebuild).
2. **Новый блок в демо** — §5 + `demo-mobile.css` + при необходимости inline в `*.html`.
3. **Новый брейкпоинт** — добавить в §2 и §7; не плодить лишние значения (предпочитать 380, 480, 640, 768, 1024, 1200).
4. **Агент Cursor** — при задачах на UI читать этот файл вместе с `planovo-design.mdc` и `Demoplan.md`.
